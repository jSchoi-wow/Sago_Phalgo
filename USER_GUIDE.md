## 탭별 사용법

---

### 📥 탭 1 — 데이터 수집

주식 데이터를 인터넷에서 내려받아 저장합니다.
**처음 실행 시 반드시 이 단계를 먼저 해야 합니다.**

#### 사용 순서

1. 기간 입력 (기본값: `2010-01-01` ~ `2024-12-31`)
2. **`데이터 수집 시작`** 버튼 클릭
3. 로그창에 진행 상황 표시됨
4. `수집 완료: 40개 종목` 메시지 확인

#### 수집되는 것

| 항목 | 내용 |
|------|------|
| 종목 수 | 삼성전자, SK하이닉스 등 40개 KOSPI 대표 종목 |
| 데이터 | 일봉 (시가/고가/저가/종가/거래량) |
| 저장 위치 | `data/cache/` 폴더 (CSV 파일) |

> 한 번 수집하면 다음부터는 캐시에서 바로 불러오므로 빠릅니다.

---

### 🧠 탭 2 — 모델 학습

AI가 주가 패턴을 학습합니다.
**데이터 수집 후에 진행해야 합니다.**

#### 사용 순서

1. 드롭다운에서 **학습할 종목 선택** (예: 삼성전자)
2. **`XGBoost + LSTM 학습`** 버튼 클릭
3. 로그창에서 학습 진행 확인 (수 분 소요)
4. 완료 후 **Feature Importance 차트** 자동 표시

#### 학습되는 모델

| 모델 | 예측 내용 | 저장 파일 |
|------|-----------|-----------|
| XGBoost | 5일 후 주가 상승/하락 확률 | `saved_models/xgboost_model.joblib` |
| LSTM | 5일 후 예상 수익률 | `saved_models/lstm_model.keras` |

#### Feature Importance 차트

- 학습 완료 후 우측에 막대 차트 표시
- **막대가 길수록** 예측에 중요한 지표
- 예: `RSI`, `MA20`, `Volume_Ratio` 등

> 한 종목으로 학습한 모델을 여러 종목 백테스트에 적용할 수 있습니다.

---

### 📊 탭 3 — 백테스트

과거 데이터로 전략을 시험해봅니다.
**모델 학습 후에 진행해야 합니다.**

#### 사용 순서

1. 종목 코드 입력 (쉼표로 여러 개 가능)
   - 예: `005930,000660,035420`
   - 삼성전자=`005930`, SK하이닉스=`000660`, NAVER=`035420`
2. **`백테스트 실행`** 버튼 클릭
3. 완료 후 성과 지표 카드 및 차트 자동 표시

#### 성과 지표 설명

| 지표 | 의미 | 좋은 값 |
|------|------|---------|
| 총 수익률 | 전체 기간 동안 번 돈 비율 | 높을수록 좋음 |
| 연간 수익률 | 1년 평균 수익률 | 10% 이상이면 우수 |
| 샤프 지수 | 위험 대비 수익 효율 | 1.0 이상이면 우수 |
| MDD | 최대 낙폭 (가장 많이 잃었을 때) | 낮을수록 좋음 |
| 승률 | 수익 난 거래 비율 | 50% 이상 목표 |
| 거래 횟수 | 총 매매 횟수 | - |

#### 포트폴리오 수익 곡선

- **청록색 실선** = 전략 수익률
- **회색 점선** = KOSPI 지수 (벤치마크)
- 전략이 점선보다 위에 있으면 시장 대비 초과 수익

#### 백테스트 조건 (기본 설정)

| 조건 | 값 |
|------|----|
| 초기 자본 | 1,000만원 |
| 종목당 최대 비중 | 10% |
| 수수료 | 0.015% (매수/매도 각각) |
| 슬리피지 | 0.1% |
| 손절 | -5% |
| 익절 | +15% |
| 학습 기간 | 2010~2020 |
| 테스트 기간 | 2021~2024 |

#### 결과 파일 저장

백테스트 완료 후 `results/` 폴더에 자동 저장됩니다.

| 파일 | 내용 |
|------|------|
| `portfolio_values.csv` | 날짜별 포트폴리오 평가액 |
| `trade_history.csv` | 전체 매수/매도 거래 내역 |
| `metrics.csv` | 성과 지표 요약 |

### 🔍 탭 4 — 종목 분석

특정 종목의 기술적 지표와 매매 신호를 시각화합니다.
**데이터 수집만 되어 있으면 바로 사용 가능합니다.**

#### 사용 순서

1. 드롭다운에서 **종목 선택**
2. **`분석`** 버튼 클릭
3. 4분할 차트 자동 표시

#### 차트 구성

```
┌────────────────────────────────┐
│  ① 가격 + 이동평균 + 매매신호  │  ← 종가, MA20, MA60, 볼린저밴드
│                                │     🔺 빨간 삼각형 = 매수 시점
│                                │     🔻 파란 삼각형 = 매도 시점
├────────────────────────────────┤
│  ② RSI                        │  ← 70 이상: 과매수(매도 고려)
│                                │     40 이하: 과매도(매수 고려)
├────────────────────────────────┤
│  ③ MACD                       │  ← MACD선이 Signal선 상향돌파 = 매수
│                                │     막대(히스토그램)로 강도 확인
├────────────────────────────────┤
│  ④ 거래량                      │  ← 주황선 = 20일 평균 거래량
│                                │     평균보다 높으면 신뢰도 상승
└────────────────────────────────┘
```

#### 매매 신호 해석

| 신호 | 조건 |
|------|------|
| 🔺 매수 | MA 정배열(MA5>MA20>MA60) + RSI 40~70 + 거래량 증가 + 주봉 상승 추세 |
| 🔻 매도 | MA 역배열 전환 or RSI 75 초과 or 볼린저밴드 상단 이탈 후 복귀 |

## 자주 묻는 질문

**Q. 데이터 수집이 느려요**
A. 최초 수집 시 40개 종목을 인터넷에서 받으므로 1~2분 걸립니다. 이후에는 캐시 사용으로 즉시 로드됩니다.

**Q. 모델 학습이 오래 걸려요**
A. LSTM 학습 때문에 3~5분 정도 소요됩니다. 로그창에서 Epoch 진행 상황을 확인하세요.

**Q. 백테스트 결과가 좋지 않아요**
A. 한 종목만 학습한 모델을 여러 종목에 적용하면 성능이 낮을 수 있습니다. 더 많은 종목으로 학습하면 개선됩니다.

**Q. 실제 매매에 사용할 수 있나요**
A. 현재 버전은 분석과 백테스트만 지원합니다. 실매매 API 연동은 추후 추가 예정입니다.
